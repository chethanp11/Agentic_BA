# Node: build-iteration-plan

## Purpose

Convert the current intent and repository state into the next iteration plan.

## Read

- `intent/product-intent.md`
- `intent/feedback.md`
- `intent/gaps.md`
- current `design/` artifacts
- current `src/` artifacts
- current `tests/` artifacts
- current `dev_log/` artifacts

## Write

- `plan/design-update.md`
- `plan/code-update.md`
- `plan/test-update.md`

## Output

- scoped design changes
- scoped implementation changes
- scoped test changes
- blockers or deferred items

## Rules

- keep the plan bounded to one iteration
- preserve unresolved questions
- do not hide contradictions
