# Node: derive-design-updates

## Purpose

Map the plan into concrete design file changes.

## Read

- `plan/design-update.md`
- `plan/code-update.md`
- `plan/test-update.md`
- relevant `design/` files

## Write

- design artifacts that actually need to change

## Output

- file-specific design deltas
- ownership decisions
- unresolved design questions

## Rules

- update the right design file only
- do not spread the same content across every design file
