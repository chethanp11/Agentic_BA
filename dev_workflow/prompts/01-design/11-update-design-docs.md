# Node: update-design-docs

## Purpose

Apply the approved design changes and capture the decision in the design log.

## Read

- `plan/design-update.md`
- derived design deltas
- current `design/` files

## Write

- `design/system-design.md`
- `design/architecture.md`
- `design/ux-flows.md`
- `design/acceptance-criteria.md`
- `dev_log/design-update-log.md`

## Output

- updated design docs
- logged design decision or gap

## Rules

- keep the design aligned with the plan
- do not encode implementation details into the design
