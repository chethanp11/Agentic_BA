# Node: fix-code-issues

## Purpose

Repair implementation defects identified by review or validation.

## Read

- fix target notes
- relevant design artifacts
- relevant `src/` files

## Write

- `src/` files
- `tests/` files when the fix requires regression coverage

## Output

- targeted code repair
- any necessary regression test update

## Rules

- keep the fix narrow
- do not change requirements to avoid a defect
