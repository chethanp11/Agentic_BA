# Node: write-closeout-log

## Purpose

Record the final state of the iteration and the next starting point.

## Read

- current plan files
- current design files
- current test files
- current validation evidence

## Write

- `dev_log/*` entries needed for closeout

## Output

- completed work summary
- remaining risks
- deferred items
- next iteration starting point

## Rules

- do not overstate completion
- keep the handoff concise and factual
