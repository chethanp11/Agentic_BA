# Node: scope-code-changes

## Purpose

Define the smallest safe implementation scope for the current iteration.

## Read

- `plan/code-update.md`
- relevant design artifacts
- current `src/` state

## Write

- implementation scope notes or code task breakdown

## Output

- file-level implementation targets
- risks and dependencies
- what is explicitly out of scope

## Rules

- do not widen the scope beyond the plan
- do not start coding without a stable target
